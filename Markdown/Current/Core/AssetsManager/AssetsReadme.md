# 资源加载系统设计文档

## 设计理念

在常规游戏开发流程中，资源加载逻辑往往分散在各个业务模块内部，资源路径硬编码、加载方式不统一、重复加载、资源管理混乱等问题普遍存在。随意的资源实例化不仅会造成磁盘IO冗余、内存资源浪费，还会导致资源生命周期不可控、引用杂乱、后期难以维护优化，严重破坏框架整体性与运行稳定性。

为统一管控游戏全部资源的加载、生成、缓存与分发流程，本框架搭建**分层管控、配置驱动、双模式加载、自带缓存复用**的资源加载系统。系统以集中化资源管控为核心目标，将所有资源获取逻辑收拢至单一模块，杜绝资源分散加载，实现游戏资源的统一调度、统一管理、统一复用。

本系统采用**总控管理层 + 工具执行层**的分层架构。资源总控层负责接收全局事件、响应外部资源请求、调度工具层执行加载逻辑，不参与底层加载细节；依托事件系统完成资源请求的接收与分发，保持与框架其他模块一致的事件驱动通信规范，保证架构风格统一。工具层封装底层加载能力，内置**同步加载、异步加载**两种主流加载方案，适配不同场景下的资源加载需求，兼顾即时生成与卡顿优化。

为彻底消除资源路径硬编码问题，本系统**完全依赖配置系统提供资源路径。**外部业务不允许直接写入资源地址，仅通过配置Key间接获取路径数据，由配置层提供标准化资源地址，再交由资源系统完成解析加载，实现资源路径外置、数据与逻辑分离。

针对频繁重复加载同类资源造成的性能损耗，系统内置资源缓存器，临时承担资源复用池职责。在资源重复请求时优先读取内存缓存，避免重复IO读取与实例化开销，减少资源冗余生成，初步实现资源复用、内存节省的管控能力。

整体而言，资源加载系统剥离了散落于业务中的资源创建逻辑，以分层结构、配置驱动、缓存复用为设计核心，为实体工厂、渲染模块、对象池等下游系统提供稳定、统一、可复用的资源供给能力。同时保持高度兼容性与拓展性，为后续完善资源引用计数、资源卸载、分级资源池、可视化资源监控预留架构空间。

处理事件，接收配置路径，选择加载方式加载资源

### 资源加载工具

设立有同步，异步加载方法，满足不同需求

暂时替代对象池，如果有重复资源优先从缓存加载

## 未来拓展

当前版本资源加载系统已完成**分层架构搭建、配置驱动加载、同步/异步双模式支持、基础资源缓存复用**的核心能力建设。统一收拢了全局资源加载逻辑，彻底解决了资源路径硬编码、加载逻辑散乱、无统一管控的传统问题，可为实体生成、场景构建、渲染输出等模块提供稳定可靠的资源供给能力。

但现阶段资源系统仍处于**功能可用、自动化与可视化不足、智能调度缺失**的基础阶段，缺少运行时监控、流程可视化、智能决策与精细化内存管控能力。未来将从**可视化工具链、智能加载策略、内存精细化管理、加载流程优化**多个方向持续迭代，向工业化、智能化资源调度体系升级。

1. **资源加载全流程可视化日志系统**

后续将搭建完善的**资源加载可视化日志体系，**记录每一份资源的加载时机、加载来源、加载耗时、请求模块、缓存命中状态、资源大小。通过编辑器窗口直观展示全局资源加载流水，实现资源加载行为可追溯、异常加载可排查、冗余加载可统计，解决当前资源加载黑盒问题。

2. **资源加载流程节点可视化编辑**

对标事件系统、配置系统的可视化工作流目标，未来将实现**资源加载流程节点可视化编排。**支持通过可视化节点配置资源加载优先级、加载依赖关系、预加载规则、场景切换资源策略，摆脱代码固定加载流程的限制，实现资源加载流程可配置、可可视化、可灵活重组。

3. **AI 智能动态加载策略决策**

引入**AI动态判断加载机制，**框架将根据当前设备性能、场景资源压力、帧耗时状态、资源类型、资源复用频率，**自动判断并切换同步/异步加载方式。**同时智能决策资源预加载、延时加载、降级加载策略，在保证画面流畅度的前提下最大化资源加载效率，实现资源调度智能化、自适应化。

4. **完善精细化资源池与内存回收机制**

现有缓存器仅实现基础复用能力，未来将升级为分级对象资源池体系，按纹理、模型、预设、音频分类管理资源池。新增资源生命周期管理、自动闲置回收、内存峰值检测、冗余资源释放机制，彻底解决内存堆积、资源残留、隐式泄漏问题。

5. **资源依赖自动解析与预加载系统**

拓展资源依赖树解析能力，自动识别资源嵌套依赖关系，实现**父资源加载自动预加载子依赖资源。**支持场景开启前、关卡切换时的全自动预加载与资源预热，大幅减少游戏运行时加载卡顿。

6. **加载管线标准化迭代升级**

统一资源加载、解析、实例化、缓存、分发、卸载全管线规范，建立完整的资源生命周期工作流。让资源系统从目前的「功能型加载工具」升级为**可监控、可配置、可智能调度、可自动化运维**的工业级资源底层调度中心，适配大型项目海量资源的迭代需求。
