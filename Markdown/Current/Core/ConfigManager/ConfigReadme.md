# 配置系统设计文档

## 设计理念

在传统游戏开发模式中，项目数值、资源路径、业务参数、实体属性等核心数据常以**硬编码形式内嵌于业务代码中。**该模式开发门槛低、实现简单，但会带来严重的工程化缺陷：数值修改需要改动代码、频繁重新编译、参数分散杂乱、路径信息写死难以维护，导致项目迭代效率低下、业务与数据高度耦合，不利于团队协作、版本管理与灵活调参。

为彻底解决**代码硬编码泛滥、数据与逻辑耦合、路径依赖模糊、配置读取方式不统一**等痛点，本框架设计实现了**基于元配置驱动的全自动泛型配置系统。**

系统核心设计思想为**数据外置、逻辑解耦、路径抽象、统一查询。**将所有游戏业务可变参数、实体配置、场景参数、业务规则全部抽离至外部配置文件，让纯代码只负责逻辑执行，不再承载任何可变数值与资源路径，实现**业务逻辑与数据配置的彻底分离。**

针对传统配置系统普遍存在的「配置路径分散、读取路径混乱、新增配置需适配新读取逻辑」问题，本架构引入**元配置顶层管理机制，**以统一的元配置索引表管理所有子配置的路径信息与类型信息，彻底解决路径模糊、配置散乱、查找困难的问题。

在此基础上，系统内置单配置读取、多同类读取、多异类读取、整目录加载、整表获取等全套泛型加载工具，封装完整的反射加载与自动解析逻辑，对外屏蔽所有文件路径、读取方式、解析细节、类型转换差异。

最终实现**外部仅需传入唯一业务Key，内部自动完成路径匹配、文件查找、反序列化、数据返回**的极简调用范式。彻底摆脱人工维护路径、手动解析配置、硬编码查表的传统开发模式，建立一套统一、稳定、可扩展、零侵入的配置数据服务体系，为全项目模块化、事件驱动、配置化动态调控提供底层数据支撑。

## 模块总览

*Unity内架构目录 ： Assets/Scripts/Core/Config*

### 配置总控

主要功能：

1. 配置查询核心
2. 元配置初始化加载所有配置的配置路径和对应的资料类（json）
3. 划分了多种读取配置方法，分别是【单配置读取、多同类读取、多异类读取、整目录加载、整表获取】
4. 加载对应的事件发布和接收子模块，与事件系统联动

### 配置加载工具

目前只定义了Json加载工具，默认加载字典格式Json，格式为{顶级键名:{中级键名:{子级键名:具体属性}}，规范配置统一按这个规范编写

### 配置资料

主要存储对应配置的资料类映射模板，让配置可以存储在指定容器里

## 接口规范

1. 先写好普通配置表

2. 写对应的配置模板容器资料

3. 定好配置路径，写入元配置

4. 根据事件值写对应的解析方法

5. 返回包含配置值的事件

## 未来拓展

当前版本配置系统已彻底解决传统开发中**数值硬编码、路径写死、数据逻辑耦合**的核心痛点，依托元配置索引体系与泛型反射加载逻辑，实现了「单Key查询、全自动寻址、无路径依赖」的标准化配置读取能力，完成了数据层与业务层的基础解耦，能够稳定支撑现有框架所有模块的配置驱动需求。

但现阶段配置系统仍存在格式单一、性能未极致、缺少可视化工作流、配置链路不可追溯的短板，整体能力仍处于**底层逻辑成型、工具生态不完善**的阶段，后续将从**多格式兼容、性能深度优化、可视化工具链、链路溯源管控**四个维度持续迭代升级。

1. **拓展多类型配置文件解析能力**

当前系统仅支持**字典结构JSON格式**的配置解析，文件格式兼容性有限，无法适配游戏项目多样化的配置编写习惯。未来将拓展解析器体系，新增Excel、通用结构化Json、表格类配置文件的自动解析、自动转译、自动序列化能力，支持策划常用表格配置一键导入项目，自动生成标准配置文件，摆脱单一JSON格式限制，适配工业化项目配置规范。

2. **重构反射加载体系，极致优化运行性能**

现阶段配置加载基于通用泛型反射实现，虽然开发通用性极强，但未做专属优化，反射调用、类型解析、实例创建存在冗余开销。后续将引入**反射树预构建机制，**在初始化阶段预缓存所有配置类型、字段映射、解析规则，构建静态反射映射树。运行时直接读取缓存映射数据，规避重复反射遍历与动态类型解析，大幅降低配置加载与读取的运行时开销，将配置系统性能打磨至工业级极致水平。

3. **搭建可视化配置编辑与自动生成工作流**

与事件系统一致，当前配置新增、修改、绑定仍依赖代码配合文件编辑，缺少低代码工具支撑。未来将开发编辑器可视化配置工具，实现**可视化便捷编写配置、拖拽式配置绑定、一键反序列化生成配置数据**的能力。打通配置与实体工厂、地图工厂、事件系统的联动生成逻辑，支持通过可视化操作直接生成业务可用配置，彻底摆脱手动编写配置文件、手动适配解析逻辑的开发模式，建立标准化、自动化的配置工作流。

4. **实现配置调用溯源与全局统计监控**

新增**配置链路追踪系统，**对所有配置的加载时机、调用模块、使用场景、引用次数进行全局统计与日志记录。通过可视化窗口直观展示当前配置被哪些系统调用、在哪些逻辑中生效、资源占用情况，解决大型项目中配置冗余、无效配置残留、配置调用链路不清晰的问题。实现配置的可监控、可溯源、可管控，进一步提升项目的可维护性与工程化规范度。

整体迭代目标：将现有「代码驱动、单格式、通用反射」的基础配置系统，升级为**多格式兼容、高性能预解析、可视化可编辑、全链路可溯源**的工业化配置生态，全面适配大型游戏项目的长期迭代需求。
