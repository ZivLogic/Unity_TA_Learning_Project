# 输入系统设计文档

## 设计理念

传统游戏输入开发常存在输入逻辑散落、按键状态判断混乱、帧抖动误触发、输入拦截规则不统一、多场景输入冲突、输入功能复用性差等问题。多数项目直接在业务逻辑中判断按键状态，导致输入逻辑与游戏业务高度耦合，难以统一管控输入优先级、场景输入权限、输入阻断规则，后期新增输入功能、修改交互逻辑成本极高，无法适配复杂、多场景、可配置的交互需求。

为解决原生输入检测不稳定、逻辑散乱、不可控、难拓展的问题，本框架自研一套**基于双帧快照检测、分层管控、可配置拦截、高工具性、标准化接口拓展**的现代化输入系统，实现游戏输入逻辑的集中化、规范化、解耦化管理。

系统核心输入判定采用**双帧原始快照对比机制。**通过逐帧记录按键与鼠标的原始输入快照，保存上一帧与当前帧的完整输入状态，通过帧间状态差分精准判定按键按下、按住、抬起、点击、双击等行为。完全规避单帧瞬时判定不稳定、帧跳变误判的传统输入Bug，让所有输入行为判定精准、稳定、可追溯。

针对游戏多场景交互冲突需求，系统设计**输入上下文管控体系**与**鼠标行为枚举配置机制。**通过上下文标识当前游戏交互环境，实现不同场景下输入规则自动切换、局部输入屏蔽、场景专属交互生效，让输入逻辑具备场景隔离能力，解决多界面、多游戏状态下的输入抢占与混乱问题。

为统一管理输入阻断、优先级覆盖、交互屏蔽逻辑，系统搭建**通用输入拦截层，**并定义统一拦截接口规范。所有自定义拦截逻辑均遵循同一接口标准实现，支持拦截优先级配置、动态开关、配置表调控，保证新增拦截规则无需改动底层输入流程，实现输入拦截能力的可拓展、可配置、可堆叠。

同时系统封装**高复用输入工具层，**集中收纳所有通用交互逻辑，包括鼠标屏幕坐标转世界坐标、鼠标选中检测、输入枚举字符串转换、鼠标光标固定、视角无限旋转矫正等通用工具能力。将高频、零散的交互算法统一封装，彻底杜绝业务层重复编写输入工具逻辑，大幅提升项目交互开发效率与逻辑统一性。

整体架构采用**管理层调度 + 工具层实现 + 拦截层管控**的分层设计，完全剥离业务层输入判断逻辑，让游戏所有输入行为统一受控、规则可配、状态稳定、拓展规范，为游戏交互、镜头操作、实体选中、UI交互、场景操作提供底层标准化输入支撑。

## 模块总览

*Unity内架构目录 ： Assets/Scripts/Core/Input*

### 输入总控

主要功能：

1. 双帧检测判断按键状态
2. 通用拦截确定输出
3. 配置化外部可更改

### 输入工具

目前功能：

1. 鼠标选中
2. 鼠标世界位置判断
3. 鼠标锁定
4. UI穿透判断
5. 层级判断
6. 输入枚举转字符串
7. 无限旋转

### 输入拦截

所有拦截层都继承规范拦截层接口，设定有上下文和输入枚举判断，配合配置优先级判断，确定输入是否是有效值

## 未来拓展

当前版本输入系统已完成**双帧快照精准检测、分层架构管控、上下文场景隔离、可配置拦截优先级、自定义按键映射、通用输入工具封装、标准化拦截接口**的完整核心能力搭建。彻底解决了传统输入逻辑散乱、判定不稳定、交互冲突、规则不可控的问题，实现了输入检测标准化、交互逻辑解耦化、拦截规则配置化，能够稳定支撑游戏所有基础交互与场景输入需求。

现阶段输入系统底层判定逻辑、按键自定义规则、拦截分层体系均已成型，但整体配置仍依赖代码初始化赋值，**已支持按键规则自定义修改，暂未实现配置文件序列化与自动加载能力，**缺少可视化编辑工作流与自动化配置链路。后续将围绕可视化编辑工具、配置反序列化自动化、输入工具链扩容、输入链路可监控四大方向持续迭代升级。

1. **可视化输入规则编辑 + 配置反序列化自动化**

对标事件系统、配置系统的可视化低代码迭代目标，未来将搭建输入系统专属编辑器窗口，支持**上下文状态管理、拦截层级优先级排序、自定义按键映射、场景输入开关规则**的可视化编辑。

依托全局配置系统实现**输入配置全链路序列化与反序列化能力：**编辑器内调整的所有按键规则、交互权限、拦截优先级，将自动生成标准化配置文件。项目运行时自动读取并还原整套输入配置，彻底摆脱代码赋值、手动配置的开发模式，实现输入规则可视化编辑、保存即生效、运行自动挂载的完整工作流。

2. **持续拓展通用输入工具链库**

基于现有鼠标选点、坐标转换、光标控制、枚举转换、视角无限旋转矫正等成熟工具能力，持续扩充输入工具层生态。新增组合键检测、长按阈值调控、输入平滑滤波、镜头回弹矫正、多状态手势识别等通用交互工具，沉淀成套高复用输入工具集，持续降低业务交互开发成本。

3. **实现输入行为日志可视化溯源**

新增全局输入快照日志系统，完整记录每帧原始输入状态、帧对比差分结果、输入触发行为、拦截生效记录、上下文切换节点。通过可视化面板直观排查输入误触发、拦截冲突、场景输入失效等异常问题，让输入判定流程全透明、可调试、可追溯。

4. **基于配置驱动的跨平台输入适配拓展**

依托完善的配置化架构，后续可通过新增配置表快速拓展手柄、触屏等多端输入适配逻辑。所有平台输入规则统一由外部配置管控，无需改动底层输入快照与拦截核心逻辑，实现一套输入架构多平台兼容拓展。

整体迭代目标：将当前**逻辑稳定、可自定义、规则可配**的成熟输入底层，升级为**可视化可编排、配置自动化、全链路可监控、跨平台可拓展**的工业化交互体系，完全契合整套框架工具化、配置化、低代码的迭代方向。
