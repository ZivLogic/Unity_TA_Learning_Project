# 框架概要

## 前言

本文档为本人**非科班出身、从零独立自研**的 Unity 可扩展游戏底层框架完整技术说明，也是个人在游戏架构设计与技术美术方向深耕过程中，对底层架构思想、系统拆解能力、工程化实现思路的完整沉淀与系统性总结。

本框架以**全局统一调度、模块无硬编码依赖、事件全链路可控、数据全配置化**为核心设计目标，历时一个半月完成核心底层全栈搭建，针对性解决传统游戏开发中模块耦合严重、事件逻辑混乱、硬编码泛滥、扩展维护成本高昂等行业常见问题，最终形成一套高解耦、高健壮、高可控、可快速迭代业务的完整底层架构体系。

框架整体采用**中央总控调度 + 分层模块化**设计思路，以总控系统作为全局唯一入口，统一管理所有子系统生命周期与执行时序；自研**多队列、强类型、可配置路由**的全链路事件系统，实现跨模块完全解耦通信；搭建**元配置驱动、反射自动加载**的数据配置层，彻底剥离数据与业务逻辑；采用**中央工厂 + 子功能工厂**双层架构完成对象实例化管理，同时配套实现标准化资源加载系统、三层架构输入系统，形成从调度、通信、数据、实例化到资源、输入的全闭环底层支撑能力。

在开发过程中，我同步配套实现 Json 配置自动生成、MD 文档版本管理等工程化效率工具，让框架从核心运行逻辑到开发工作流均具备规范化、可迭代、可维护的工业级特性。本框架并非一次性demo产物，而是一套可直接支撑完整游戏项目落地、可持续扩展业务系统、可深度对接渲染管线与TA相关能力的稳定底层基座。

本文档仅包含**前言、框架概述、架构目录、项目链接**四个核心基础模块，完整呈现框架设计思想、整体结构、模块划分与项目溯源。后续所有子模块详细设计、实现原理、使用规范、迭代日志，均统一归入架构目录下分章节展开，保持文档结构简洁清晰、重点突出，完整呈现这套自研框架的设计理念与技术价值。

## 框架概述

本项目为**非科班从零自研、基于Unity实现的可扩展、高解耦、配置驱动型游戏框架，**历时一个半月完成核心底层架构全栈搭建，以「全局统一调度、模块无硬编码依赖、事件全链路可控、数据全配置化」为核心设计目标，彻底规避传统游戏开发中模块耦合、事件混乱、硬编码泛滥、扩展成本高的行业痛点，构建了一套具备工业级健壮性、可快速迭代新增业务的完整底层架构体系。

框架整体采用**中央总控调度+分层模块化设计**，以**游戏总控系统**为全局唯一核心入口，统一完成所有核心子系统的初始化、生命周期管理与全局事件调度，实现框架启动与运行的全流程可控。核心通信层采用自研**分层多队列强类型事件系统，**针对逻辑、渲染、物理、音频四大游戏核心执行维度，分别设计独立协程队列与精细化帧率控制，彻底分离不同优先级任务的执行时序，避免线程阻塞与执行冲突；针对传统object[]事件传参存在的隐性协议、顺序依赖、类型不安全问题，完成两轮技术迭代：先落地**强类型事件基类体系，**统一事件传输规范，再自研**柔性万能数据打包容器，**实现基于字符串键值的任意类型、无限数量数据动态存储与安全读取，最终形成「强类型事件壳+柔性安全传参容器」的最优通信方案，新增事件仅需定义空壳类即可完成扩展，零传参代码冗余；同时通过静态常量字符串类彻底消除魔法字符串，提升代码可维护性。针对事件链硬编码、动态管控困难的痛点，基于**反射机制+双层配置路由表**实现事件全链路动态调度，通过配置层控制事件启用状态、全局缓存层实现事件运行时动态开关，所有模块通过「事件发布」「事件接收」两大标准化基类接口接入事件系统，实现模块间零直接依赖、完全解耦通信。

数据配置层自研**元配置驱动的泛型反射加载系统，**彻底摒弃传统配置硬编码、路径耦合、查询繁琐的问题，通过元配置文件实现配置资源自发现、自加载，支持单配置、多同类配置、多异类配置、目录批量加载、配置表全量读取等全场景查询需求，外部业务仅需传入配置Key即可完成数据获取，完全屏蔽配置路径与加载细节，实现数据层与业务层的彻底解耦。

对象实例化层采用**中央总控工厂+子功能工厂双层架构**，中央工厂作为全局调度中枢，负责事件响应、工厂管理与实例化调度，各子工厂通过标准化接口完成功能实现，已落地逻辑工厂、实体工厂、地图工厂三大核心子工厂，分别实现实体逻辑脚本动态挂载、实体配置化生成与标签管理、地图数据预计算与前置配置功能，所有工厂遵循统一接口规范，扩展新工厂无需修改中央工厂代码，符合开闭原则。

同时框架配套搭建**同步/异步双模式资源加载系统，**支持基于配置路径的全类型资源加载，结果直接对接实体、渲染、对象池等下游系统，实现资源加载全流程自动化；自研**三层架构输入系统，**划分为管理层、工具层、拦截层，通过原始数据快照实现双帧输入检测，支持多版本兼容、上下文绑定与通用输入拦截，所有输入规则可通过配置系统动态调控，适配复杂游戏输入逻辑需求。

此外，框架配套开发Json配置自动生成器、MD文档版本管理更新器等效率工具，实现开发流程自动化、文档管理规范化，整体架构具备高可扩展性、高可维护性、高运行稳定性，可无缝对接后续游戏业务逻辑开发，为完整游戏项目落地提供了坚实、灵活的底层支撑。

## 设计原则与规范

本框架整体设计严格遵循以下核心原则：

- **单一职责：**每个模块只负责固定核心能力，边界清晰

- **低耦合高内聚：**模块间仅通过事件与配置通信，无硬编码依赖

- **开闭原则：**扩展功能无需修改底层核心代码

- **配置驱动：**逻辑可控、数据可配置、流程可动态调度

- **全局统一调度：**生命周期、执行时序、事件路由全部中心化管理
