# Event-Driven Chess Framework
*基于事件总线的棋子游戏框架，通过5条独立协程管道（逻辑/渲染/物理/音频/输入）实现模块解耦，验证系统边界设计与事件驱动架构的闭环可行性。*

***

## ✨ 核心设计

- **事件系统：**万能数据容器 + 空壳强类型事件，新增事件值无需修改字段；5条独立管道支持帧率独立控制
- **配置系统：**元配置自举 + 反射解析，支持单/多/异类/批量配置获取
- **输入系统：**双帧检测 + 上下文拦截层，同一按键自动区分不同页面功能
- **工厂系统：**中央工厂 + 子工厂（地图/实体/逻辑/物理/渲染），实体组件按需挂载
- **资源加载：**同步/异步 + 缓存器，优化加载性能

## 📁 模块结构

| 系统        | 职责                        |
| ----------- | --------------------------- |
| 事件系统    | 5条管道，协程并行，帧率独立 |
| 配置系统    | 元配置自举，反射解析        |
| 工厂系统    | 中央调度 + 6个子工厂        |
| 输入系统    | 双帧检测 + 拦截层           |
| 实体/ID系统 | 生命周期管理 + 全局身份校验 |
