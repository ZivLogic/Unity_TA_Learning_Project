# Sensing Systems

> 当交互界面从键盘鼠标推进到生理、情绪与神经信号时，游戏系统应该如何重新设计？

这个文件夹记录的是我在自学游戏开发之外，对几个底层问题的长期推演。它们不是零散的“杂谈”，而是围绕同一个命题展开的三次架构实验：

| 设计         | 核心问题                                           | 跨领域依赖                          |
| ------------ | -------------------------------------------------- | ----------------------------------- |
| 反作弊系统   | 如何区分“人”与“完美机器”？                         | 神经科学、博弈论、心理学、统计学    |
| 情感 NPC     | 如何让 AI“感知并记住”情绪的多维面貌？              | 心理学、复杂系统、遗忘/成长曲线建模 |
| 脑机游戏平台 | 当输入变成神经电信号，玩法本身能否是“情绪的叠加”？ | 神经医学、计算机架构、伦理设计      |

## 为什么会有这个文件夹？

我是非科班自学者。在做游戏作品集的过程中，我发现很多“看似不属于游戏”的问题——作弊检测、情感模拟、人机交互边界——其实共享同一套系统设计思维。

这里没有完整的代码实现。有的是：
- 对问题边界的拆解
- 跨领域知识的迁移路径
- 分阶段的落地推演
- 以及一个自学者的思考痕迹

## 怎么阅读？

每个子文件夹是一份独立的概念文档，按时间线记录了我从“发现问题”到“形成方案”的完整推演。如果你只想快速了解核心思路，建议从 [反作弊系统设计](Anti-Cheat Architecture/Anti-Cheat-Blueprint.md)开始——它是目前推演最完整的一个。
